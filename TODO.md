## LLM-ассистент прямо на дашборде

Кликаешь по цифре / сущности / строке в таблице → задаёшь вопрос своими
словами, под капотом LLM вытаскивает из данных контекст и отвечает.

**Типовой сценарий:**

> На неделе 13–19 апр Bybit вылез на #4 в платформах. Почему?

Ассистент под капотом:

1. Лезет в `community-brain/wiki/pages/platforms/bybit.md` — читает
   summary, `source_ids`, `related`.
2. По `source_ids` вытягивает сами сообщения из
   `community-brain/data/messages_2026-04-1X.json`.
3. Если в wiki есть граф связей (через `[[wikilinks]]`) — прыгает по
   связанным сущностям: кто её упомянул, в каком контексте, что ещё
   обсуждалось в этих же тредах.
4. LLM получает {карточка + N сообщений-источников + связанные
   сущности} и отвечает в духе «в этот период OpenAI анонсировала
   интеграцию с Bybit для выплат исследователям, это подняло упоминания
   X3 vs прошлой недели» — с цитатами из сообщений.

**Что уже есть:**

- `community-brain/search_wiki.py` — трёхслойный гибридный поиск
  (ChromaDB + BFS по графу + LLM-синтез). Это ровно тот движок, который
  нам нужен — можно дёрнуть как функцию, не переизобретать.
- `community-brain/wiki/overview.md` — LLM-синтез всей базы, хорошо
  подходит для grounding'а в общий контекст.
- `community-brain/bot.py` — уже умеет отвечать на вопросы к базе
  знаний, только через Telegram-бот интерфейс. Логика ответа
  переиспользуема.

**Что надо добавить:**

- Лёгкий бэкенд-endpoint (или serverless) который принимает
  `{entity, question, context_hint}` и возвращает markdown-ответ.
  Self-contained HTML-дашборд сам в LLM ходить не может — нужен
  сервер. Либо локальный хостинг на том же компе, что и
  community-brain (и так всё рядом), либо GitHub Pages + Workers.
- UI: сбоку / снизу выезжающая панель «спросить ассистента», с
  подстановкой контекста (выбранная сущность / неделя / категория).
  Fetch → streaming response → markdown.
- Кеширование ответов по (entity, week) — часто будут задавать
  похожие вопросы, дешевле не дергать LLM каждый раз.

**Ограничения / чего остерегаться:**

- Текущая архитектура ai-pulse — строго self-contained HTML,
  zero-runtime-fetch. Фича ломает этот принцип (нужен внешний
  endpoint), так что возможно имеет смысл сделать «pro»-версию
  дашборда рядом со статической.
- API-ключи: если дёргаем LLM напрямую с клиента — ключ утечёт.
  Только через серверный проксик.
- Скорость: гибридный поиск через ChromaDB + LLM — 2–5 секунд на
  запрос. Нужен явный loading-state в UI.
